# PRODUCT SPEC (APPEND-ONLY)

ВНИМАНИЕ: Этот файл append-only.

- Нельзя удалять существующие строки.
- Нельзя переписывать историю.
- Можно только добавлять новые блоки в конец или добавлять блок `Изменено на...` рядом с целевым разделом.

## 1. Назначение продукта

Develop A Habit — приватный Telegram-бот для трекинга привычек и дневника.

Ключевые цели:
- Видимый прогресс на мелких шагах.
- Разделение привычек на позитивные и негативные.
- Календарная навигация с фокусом на текущую неделю.
- Ведение дневника с текстовыми и голосовыми записями.
- Поиск по дневнику, включая поиск голосовых через транскрибацию.
- Экспорт дневника и статистики.

## 2. Приватность

- Все данные строго привязаны к Telegram user id владельца.
- Операции чтения/записи всегда фильтруются по владельцу.
- Публичных страниц с персональными данными нет.

## 3. Привычки

### 3.1 Типы привычек
- Позитивные: нужно выполнить.
- Негативные: нужно не нарушить.

### 3.2 Временные слоты
- `morning` (утро)
- `day` (день)
- `evening` (вечер)

### 3.3 Расписания
- Каждый день.
- Через день.
- Конкретные дни недели.

### 3.4 Интерфейс списка привычек
- По умолчанию показывается ближайший слот.
- Есть переключатели: утро, день, вечер, все.
- Есть inline CRUD: добавить, переименовать, удалить.

### 3.5 Отметки
- `✅` выполнено/успех.
- `❌` не выполнено/провал.
- Есть отмена последней отметки.

## 4. Выходные и сверхплан

- Пользователь задает несколько выходных дней в неделе и/или точечные даты.
- В выходной привычки можно делать по желанию.
- Выполнение в выходной попадает в `extra_slots`.
- `plan_completion = completed_slots / plan_slots * 100`.
- `over_completion = (completed_slots + extra_slots) / plan_slots * 100`.

## 5. Календарь

- Календарь открывается текущей неделей.
- Есть листание по неделям.
- Можно открыть конкретный день и увидеть:
  - список привычек и статусы;
  - заметки дневника за день.

## 6. Дневник

### 6.1 Текст
- Текстовые записи добавляются в дневник и привязываются к дню.

### 6.2 Голосовые
- Голосовые сохраняются с telegram file_id.
- Для голосовых ведется транскрибация.
- Транскрибация хранится в БД.

## 7. Поиск заметок

- Отдельная команда поиска.
- Поиск идет по тексту заметки и по транскрибации голосовых.
- Есть режим `только голосовые`.
- При совпадении голосовой заметки бот отправляет соответствующее voice-сообщение.

## 8. Экспорт

### 8.1 Экспорт дневника
- Команда экспорта формирует ZIP.
- В архиве:
  - Markdown файлы по датам;
  - media файлы голосовых;
  - транскрипции в теле markdown.

### 8.2 Экспорт статистики
- Генерируется статический HTML-отчет.
- Содержит summary, месячную разбивку и недельный тренд.

## 9. Политика изменения функций

Если функция или правило меняется, рядом с соответствующим разделом добавить запись:

`Изменено на: <новое поведение>. Причина: <почему>. Коммит: <hash>.`

Пример для будущих правок:
- Изменено на: новый алгоритм недельного окна. Причина: ускорение навигации. Коммит: abc1234.

## История изменений (append-only)

- Изменено на: стартовый экран переведен на inline-меню с быстрыми переходами по ключевым разделам. Причина: требование основного интерфейса через inline-кнопки. Коммит: abc0c6f.
- Изменено на: добавлен экран `/settings` для выбора нескольких выходных дней недели. Причина: пользователь должен сам задавать число выходных в неделю. Коммит: abc0c6f.
- Изменено на: исправлена базовая миграция enum-типов, чтобы исключить двойное создание типов в PostgreSQL. Причина: ошибка миграции при первичном разворачивании. Коммит: 0cf872d.
- Изменено на: для PostgreSQL в базовой миграции используется `postgresql.ENUM` с `create_type=False`. Причина: корректный контроль создания enum через `checkfirst` и отсутствие дублей. Коммит: 3681110.
- Изменено на: в миграции дневника enum `diary_entry_type` переведен на `postgresql.ENUM(..., create_type=False)`. Причина: устранение повторного `CREATE TYPE` при разворачивании. Коммит: cd11194.
- Изменено на: главное меню сокращено до ключевых пунктов с подменю `Аналитика` и `Еще`; вторичные действия спрятаны внутрь подменю. Причина: уменьшить перегрузку главного экрана. Коммит: 86bbc4d.
- Изменено на: добавлена кнопка `⬅️ Назад` во все основные разделы и рабочие экраны. Причина: явная навигация без использования команд. Коммит: 86bbc4d.
- Изменено на: экспортные действия переведены на кнопочный сценарий (без необходимости вводить команды). Причина: button-only UX. Коммит: 86bbc4d.
- Изменено на: ускорен отклик календаря за счет пакетного расчета индикаторов недели и раннего `callback.answer()`. Причина: уменьшить задержку на нажатиях. Коммит: 86bbc4d.
- Изменено на: идентификатор Telegram пользователя хранится в БД как `BIGINT` вместо `INTEGER`. Причина: `INTEGER` переполнялся на реальных Telegram ID и ломал обработку кнопок. Коммит: 4e8c5b9.
- Изменено на: стартовые command-фильтры удалены, вход в меню выполняется через текстовое сообщение (включая `/start` как обычный текст). Причина: интерфейс должен быть только кнопочный без command-зависимости. Коммит: 4e8c5b9.
- Изменено на: главное меню сокращено до трех глобальных разделов `Привычки`, `Дневник`, `Настройки`, а календарь/поиск/экспорт/статистика перенесены внутрь `Настройки`. Причина: убрать перегрузку главного экрана и сгруппировать похожие функции. Коммит: aae24cd.
- Изменено на: добавлен слот привычки `Весь день` (`all_day`) для сценариев, где действие актуально весь день (особенно негативные привычки). Причина: поддержка привычек без привязки к отдельному времени суток. Коммит: aae24cd.
- Изменено на: нажатие на кнопку привычки сразу отмечает ее выполненной (`✅`), а не открывает редактирование. Причина: ускорить ежедневное отмечание и уменьшить количество шагов. Коммит: aae24cd.
- Изменено на: исправлено сохранение новых привычек и других enum-полей без падения на Postgres enum-литералах. Причина: ORM отправлял enum-имена (`NEGATIVE`) вместо значений (`negative`), что вызывало ошибку при создании привычки. Коммит: 8f01757.
- Изменено на: рабочее поле привычек очищено от сервисных действий (добавление/редактирование/удаление/откат), управление привычками вынесено в `Настройки`. Причина: минимизировать шум в ежедневной рутине. Коммит: 2dd8903.
- Изменено на: в привычках добавлена обязательная маркировка группы времени (`🌅/☀️/🌙/🗓️`) и персональный emoji привычки, задаваемый при создании. Причина: быстрый визуальный разбор списка. Коммит: 2dd8903.
- Изменено на: выходные настраиваются по конкретным датам в режиме недели или месяца, без привязки к фиксированным weekday-шаблонам. Причина: выходные могут быть плавающими и разными каждую неделю. Коммит: 2dd8903.
- Изменено на: дневник при просмотре списка за день автоматически отправляет сохраненные голосовые как voice-сообщения для прослушивания. Причина: сделать воспроизведение ГС очевидным без поиска по ID. Коммит: 2dd8903.
- Изменено на: экспорт дневника/HTML в callback-потоке использует `callback.from_user.id`, а не автора сообщения бота. Причина: исправить выгрузку `0 записей` из-за ошибочного user context. Коммит: 2dd8903.
- Изменено на: добавлена еженедельная сводка в конце недели с кнопкой запроса комментария; комментарий сохраняется в дневник (текст/голос). Причина: регулярная рефлексия прогресса и сохранение итогов недели. Коммит: 2dd8903.
- Изменено на: идентификатор ревизии миграции `0007` укорочен до формата, совместимого с таблицей `alembic_version`. Причина: развертывание падало из-за превышения длины `version_num` при миграции. Коммит: 8f41fd1.
- Изменено на: long voice транскрибация выполняется асинхронно через фоновую очередь, а пользователь получает быстрый ответ сразу после сохранения ГС. Причина: исключить зависание интерфейса на долгой транскрибации. Коммит: c0e2e51.
- Изменено на: редактирование сообщений через inline-коллбеки защищено от ошибки `message is not modified` без падения сценария. Причина: устранить хрупкость UI при повторных нажатиях. Коммит: c0e2e51.
- Изменено на: для спортивных привычек включается линейная прогрессия по повторам (база + недельный шаг), а в рабочем списке отображается текущая цель в формате `[подходы x повторы]`. Причина: поддержать постепенный рост нагрузки без потери истории. Коммит: a45bb51.
- Изменено на: при отметке `✅` спортивной привычки бот запрашивает фактические повторы по подходам (пример `10,10,8`) и сохраняет факт вместе с целевыми значениями дня. Причина: фиксировать частичное выполнение внутри тренировки, а не только бинарный чек. Коммит: a45bb51.
- Изменено на: для спорт-привычек подтверждение выполнения переведено в бинарный режим `смог придержаться плана / не смог` (inline `✅/❌`) вместо обязательного ввода повторов по подходам. Причина: упростить ежедневную отметку и снизить трение в рутине. Коммит: 61e6c4b.
- Изменено на: в `Настройки -> Управление привычками` добавлен экран `🏋️ Настроить спорт-план` с редактированием базы `NxM`, шага прогрессии и переключателем прогрессии `ВКЛ/ВЫКЛ`. Причина: пользователь должен гибко корректировать стандартные значения и прогрессию без перегруза рабочего экрана. Коммит: 61e6c4b.
- Изменено на: при изменении базового плана спорт-привычки `sport_start_date` сбрасывается на текущую дату для корректного пересчета линейной прогрессии от новой базы. Причина: избежать скачков целевых повторов после ручной корректировки базы. Коммит: 61e6c4b.
- Изменено на: в периодической статистике и HTML-отчете добавлен отдельный учет суммарных повторов `подтягиваний` и `отжиманий`. Причина: пользователь просил видеть силовой объем по ключевым упражнениям отдельно от общих метрик выполнения. Коммит: 61e6c4b.
